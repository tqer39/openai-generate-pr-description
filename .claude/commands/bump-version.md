# SemVer バージョン自動更新

変更内容を分析し、`pyproject.toml` の SemVer バージョンを適切に更新する。

## 手順

1. `git diff --cached` でステージされた変更を取得する。ステージされた変更がない場合は `git diff` で未ステージの変更を取得する。
2. `pyproject.toml` の `version` フィールドから現在のバージョンを読み取る。
3. 変更内容を分析し、以下の基準でバンプタイプを判定する:
   - **major**: 破壊的変更（`action.yml` の入力パラメータ削除・名称変更、出力形式の変更など、既存ユーザーの利用を壊す変更）
   - **minor**: 新機能追加（新しい入力パラメータ、新しいオプション、機能拡張など、後方互換性のある追加）
   - **patch**: バグ修正、リファクタリング、依存更新、ドキュメント修正、CI/CD 設定変更、テスト追加など
4. 判定結果（バンプタイプと理由）をユーザーに提示し、確認を得る。ユーザーが別のバンプタイプを希望した場合はそれに従う。
5. 以下のファイルのバージョンを新しいバージョンに一括更新する（Edit ツールを使用）:
   - `pyproject.toml` の `version` フィールド
   - `README.md` のバージョン参照例（例: `@v2.0.0` → `@v2.0.1`）
   - `docs/README.ja.md` のバージョン参照例
6. 更新後のバージョンと更新したファイル一覧をユーザーに報告する。
